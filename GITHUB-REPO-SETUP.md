# GitHub repository setup (professional look)

Use this to fill in the **About** section and **Releases** so the repo looks complete and professional (like [react-native-nitro-ota](https://github.com/riteshshukla04/react-native-nitro-ota)).

---

## 1. About section (right sidebar)

On the repo page, click the **gear icon** next to “About” and set:

| Field | Suggested value |
|-------|------------------|
| **Description** | `High-performance OTA updates for React Native via Nitro/JSI. Ship JS updates without App Store review.` |
| **Website** | `https://www.npmjs.com/package/react-native-nitro-update` (optional) |
| **Topics** | Add: `react-native`, `ota`, `over-the-air`, `nitro`, `jsi`, `updates`, `javascript` |

Then click **Save changes**. GitHub will show the description and topics in the sidebar and improve discoverability.

---

## 2. First release

Creating a release makes the repo look maintained and gives users a clear version to reference.

1. Go to **Releases** → **Create a new release**.
2. **Choose a tag:** Create tag `v0.0.1` (or `v0.0.2` if you’ve already published a new version to npm).
3. **Release title:** e.g. `v0.0.1` or `Release 0.0.1`.
4. **Description:** e.g. “Initial release. Install with `npm install react-native-nitro-update`. See README and INTEGRATION.md for setup.”
5. Click **Publish release**.

After that, the sidebar will show “Releases 1” and “Latest” with your version.

---

## 3. Optional: GitHub social image

For a nicer preview when the repo is shared (e.g. Twitter, Slack), you can add a social preview image:

- **Settings** → **General** → **Social preview** → upload an image (e.g. 1280×640 px with the project name and a short tagline).

---

## Summary

| Step | Where | What to do |
|------|--------|------------|
| About | Repo front page → gear next to “About” | Add description, optional website, topics |
| Release | **Releases** → **Create a new release** | Tag `v0.0.1`, title + short description, Publish |

Once these are done, the repo will show a clear description, topics, “Readme”, “License”, “Code of conduct”, “Contributing”, and “Releases” in the About area.
